# Планирование: Бот-Генератор Каруселей (Carousel Factory)

## 1. Концепция
Бот, который принимает от пользователя текст и/или фото, и генерирует на их основе стильную карусель (набор изображений для поста в Instagram, Telegram, LinkedIn).
В будущем бот будет монетизироваться, поэтому сразу закладываем архитектуру лимитов, триал-периодов и проверки подписок (force subscribe).

## 2. Основной функционал (MVP)
1. **Генерация:**
   - Пользователь отправляет текст (например, лонгрид или пост).
   - Бот с помощью AI (Gemini или Claude) разбивает текст на логические части (до 10 слайдов), выделяя заголовки и короткий текст.
   - Пользователь отправляет (или бот генерирует/использует стоковый) фон/фото.
   - Бот рендерит картинки: накладывает текст, плашки, логотипы, номера страниц на фон. Возвращает zip-архив или группу фото (MediaGroup) в ТГ.
2. **Лимиты и монетизация:**
   - Free Trial: 7 дней с момента `/start`.
   - Daily Limit: 10 картинок (слайдов) в день на бесплатном тарифе.
   - Обязательная подписка: перед использованием бот проверяет, подписан ли юзер на каналы спонсора/создателя.

## 3. Технический стек
- **Язык:** Python
- **Telegram Framework:** `aiogram` (асинхронный, быстрый, удобный для стейтов).
- **База данных:** SQLite (для MVP) с плавным переходом на Supabase/PostgreSQL.
  - Таблица `users`: `tg_id`, `created_at`, `trial_ends_at`, `subscription_status`.
  - Таблица `usage`: `tg_id`, `date`, `images_generated`.
- **Генерация картинок:** `Pillow` (PIL) для отрисовки текста, плашек, градиентов поверх фото.
- **AI обработка текста:** Gemini 1.5 Flash (быстро, дешево, хорошо работает с текстом).

## 4. Архитектура проекта
- `carousel_bot/`
  - `bot.py` (точка входа, роутеры aiogram).
  - `db.py` (подключение к SQLite/Supabase, функции создания юзеров, проверки лимитов).
  - `image_generator.py` (логика Pillow для обрезки, затемнения, наложения текста).
  - `text_processor.py` (обращение к Gemini API для разбивки текста на слайды).
  - `middlewares/` (middleware для проверки подписки на каналы и лимитов перед каждым сообщением).
  - `assets/` (шрифты, шаблоны, плашки).
