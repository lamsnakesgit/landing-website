_# План реализации Telegram-бота для Story Factory

Чтобы система работала "без компьютера", нам нужно перенести её с вашего локального ПК на сервер и добавить интерфейс Telegram.

## Архитектура
1.  **Сервер**: Виртуальная машина (VPS) с установленным Docker и Playwright.
2.  **База данных**: Supabase или PostgreSQL для хранения ваших настроек и истории сторис.
3.  **Очередь задач (Celery/Redis)**: Генерация видео занимает время (2-5 минут), поэтому бот должен работать асинхронно.

## Схема работы
1.  Вы пишете боту: `/generate Продать курс по ИИ`.
2.  Бот отправляет запрос в **Gemini API**.
3.  Gemini возвращает JSON-план.
4.  Бот ставит задачу в очередь.
5.  **Worker** (серверный скрипт) запускает Playwright, заходит в Google Vids, делает видео и скачивает его.
6.  Бот присылает вам готовый файл или ссылку на скачивание.

## Что нужно сделать (Шаги)
1.  **Регистрация бота**: Создать бота через `@BotFather`.
2.  **Backend**: Написать код на `aiogram` или `python-telegram-bot`, который будет вызывать наш `StoryOrchestrator`.
3.  **Docker**: Упаковать всё в контейнер, чтобы Playwright работал на сервере (нужны зависимости для браузера).
4.  **Сессия Google**: Загрузить `google_auth.json` на сервер, чтобы бот мог заходить в ваш аккаунт.

## Пример команды для бота
> **Вы**: /story Цель: Продать консультацию. Аудитория: Эксперты. Стиль: Минимализм.
>
> **Бот**: Принято! Генерирую план сторис по методу Тимочко...
> **Бот**: План готов (5 слайдов). Начинаю создание видео в Google Vids. Это займет около 3 минут.
> **Бот**: [Видео файл 1.mp4]
> **Бот**: [Видео файл 2.mp4] ...
