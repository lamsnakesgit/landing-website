# План: AI notebook / mind map tool по промпту

## Цель

Подобрать лучший инструмент не просто для хранения заметок, а для **раскрытия темы по промпту** с выводом в виде **кастомных mind maps**.

## Что именно нужно от инструмента

Нужен не обычный notebook, а система, которая умеет:

1. Принимать тему или промпт.
2. Раскрывать тему по уровням: главная идея → ветки → подтемы → детали.
3. Строить mind map не только из загруженных файлов, но и **из самого запроса**.
4. Позволять дорабатывать структуру под задачу:
   - контент-план;
   - стратегия продукта;
   - декомпозиция воронки;
   - сценарии видео;
   - карта идей для агентов / автоматизаций.

---

## Лучший вариант под задачу

### 1. Основной выбор: **NotebookLM + внешний mind map слой**

**Зачем:**

- NotebookLM хорошо раскрывает тему, суммирует материалы, вытаскивает инсайты и делает структуру из источников.
- Но сам по себе он **не является лучшим native-инструментом для кастомных mind maps по промпту**.
- Поэтому оптимальная связка:
  - **NotebookLM** = движок анализа / раскрытия темы;
  - **mind map tool** = визуализация структуры.

**Когда подходит:**

- если нужно исследовать тему по PDF, Google Docs, заметкам, ссылкам;
- если нужно быстро получить summary, FAQ, outline, key ideas;
- если нужен полу-исследовательский режим перед созданием mind map.

**Ограничение:**

- mind map по промпту там не является главным продуктовым сценарием;
- кастомную карту под формат "раскрой тему как стратегию / как курс / как pipeline" удобнее делать через отдельный LLM prompt + экспорт в карту.

---

### 2. Практически лучший вариант под твою задачу: **LLM + Mermaid/XMind/Markmap**

Это лучший путь, если цель — **делать mind maps именно по промпту**, а не просто хранить заметки.

Схема:

1. Ты даёшь промпт.
2. LLM строит дерево мысли в структурированном виде.
3. Результат сразу переводится в один из форматов:
   - `Mermaid mindmap`
   - `Markmap Markdown`
   - `XMind-compatible outline`
   - JSON-структура для дальнейшего рендера.

**Почему это сильнее обычного notebook:**

- полная кастомизация структуры;
- можно задавать стиль раскрытия;
- можно генерировать карты под конкретный use case;
- легко автоматизировать в n8n / Python / Streamlit.

## Рекомендованный стек

### Вариант A — быстрый и без разработки

- **NotebookLM** — для анализа источников;
- **XMind** или **MindMeister** — для ручной/полуавто визуализации;
- **ChatGPT / Gemini / Claude** — для генерации структуры карты по промпту.

**Итог:** быстро стартовать, но часть работы руками.

### Вариант B — лучший для твоего проекта

- **Gemini 2.5 Pro** или **Claude** — генерирует структуру карты;
- **Markmap** — визуализирует mind map из Markdown;
- **Mermaid** — если нужен простой portable-формат;
- **Streamlit** — интерфейс внутри текущего проекта;
- **n8n** — автоматизация пайплайна.

**Итог:** это уже не "ноутбук", а твой собственный генератор карт по промпту.

## Зачем это тебе

Под твою текущую экосистему инструмент даст 5 прямых сценариев:

### 1. Раскрытие темы для контента

Промпт:

> Раскрой тему "AI media factory for small business" как mind map: боли, офферы, воронки, контент, автоматизация, продажи.

На выходе:

- структура контент-стратегии;
- темы для Reels / Shorts / Telegram / YouTube;
- идеи лид-магнитов;
- логика продаж.

### 2. Декомпозиция AI-продукта

Промпт:

> Разложи продукт "AI video cloning studio" на модули: acquisition, pipeline, generation, QA, delivery, monetization.

На выходе:

- roadmap;
- архитектурная карта;
- список модулей для разработки.

### 3. Mind map для клиентских брифов

Промпт:

> Построй mind map для клиента beauty niche: offer, audience, pain points, creatives, funnel, automation.

На выходе:

- понятная карта для созвона;
- быстрый пресейл-документ.

### 4. Планирование агентов и автоматизаций

Промпт:

> Построй карту AI-агентов для content factory: researcher, script writer, image prompt agent, video agent, publisher, analyst.

На выходе:

- схема ролей агентов;
- связи между узлами;
- база для n8n/ботов.

### 5. Раскрытие сложной идеи в курс / методологию

Промпт:

> Преврати тему "AI content automation" в карту курса: beginner, intermediate, advanced, templates, cases, mistakes.

На выходе:

- модульная структура курса;
- основа для PDF / Notion / презентации.

## Как это будут использовать

### Базовый сценарий использования

1. Пользователь пишет промпт.
2. Система определяет тип карты:
   - strategic;
   - educational;
   - content;
   - funnel;
   - product architecture.
3. LLM строит иерархию 3–5 уровней глубины.
4. Система рендерит это как mind map.
5. Пользователь уточняет:
   - сделать глубже;
   - сузить тему;
   - раскрыть коммерческую часть;
   - добавить примеры;
   - преобразовать в roadmap / таблицу / сценарий.

## Что скачать / установить

Если нужен **реально рабочий стек**, рекомендую такой порядок:

### Минимум

1. **XMind** — для удобного просмотра и ручной доработки карт.
2. **Markmap** — если хочешь быстрый auto-render из Markdown.
3. **NotebookLM** — если нужен режим анализа источников.

### Если делать в текущем проекте

Скачивать отдельный "open notebook" не обязательно.

Лучше собрать своё решение внутри проекта:

- prompt input;
- выбор типа карты;
- генерация JSON/Markdown;
- рендер через Markmap/Mermaid;
- экспорт в PNG/PDF/Markdown.

## Ограничения и правда по продуктам

### NotebookLM

Сильный для:

- research;
- summarization;
- Q&A по источникам;
- разложения материалов.

Слабее для:

- кастомных mind maps как основного output;
- полного контроля формата карты;
- продуктового pipeline под твой кейс.

### XMind / MindMeister

Сильные для:

- красивой визуализации;
- ручного редактирования.

Слабее для:

- AI-раскрытия темы с нуля без внешней LLM.

### LLM + Markmap / Mermaid

Сильные для:

- генерации по промпту;
- кастомных шаблонов;
- автоматизации;
- интеграции в твой pipeline.

Слабее для:

- "из коробки" UX, если не собрать интерфейс.
