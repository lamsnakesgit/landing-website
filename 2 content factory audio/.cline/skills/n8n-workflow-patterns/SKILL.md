# n8n Workflow Patterns (Content Factory Audio)

> Этот skill помогает строить надежные и масштабируемые автоматизации в n8n, которые являются ядром "Content Factory".

## Основные сценарии проекта
- **Media Processing:** Прием фото/видео -> Анализ (Gemini) -> Генерация (VEO/Suno) -> Отправка.
- **Lead Gen:** Сбор данных -> Обогащение через AI -> Запись в Supabase.
- **Suno/Music:** Генерация текста песни -> Отправка в Suno API -> Ожидание результата -> Уведомление.

---

## Рекомендуемые паттерны
1. **Webhook → Normalize → Process → Respond:**
   - Всегда нормализуй входящие данные из Telegram (extract `chat_id`, `file_id`).
   - Используй `Respond to Webhook` ноду для мгновенного подтверждения в Telegram.
2. **Error Handling (Trap Node):**
   - Используй `Error Trigger` для логирования ошибок в Supabase или отправки уведомления админу.
3. **Wait & Poll:**
   - Для долгих генераций (Suno/VEO) используй паттерн с ожиданием или периодической проверкой статуса через `HTTP Request`.

## Правила работы с шаблонами
- Шаблоны хранятся в `templates/` в формате JSON.
- При редактировании JSON через Cline, всегда проверяй валидность структуры.
- Используй переменные окружения n8n для хранения API ключей.

## Чек-лист для самопроверки
- [ ] Workflow разделен на логические блоки (Input, Logic, Output).
- [ ] Есть обработка ошибок (Error Path).
- [ ] Данные нормализованы перед передачей в AI ноды.
- [ ] JSON шаблон валиден и готов к импорту.
