# Systematic Debugging (Content Factory Audio)

> Этот skill используется, когда система ведет себя непредсказуемо (ошибки API, сбои рендеринга, проблемы с базой данных).

## Специфика отладки в проекте
- **AI APIs:** Проверяй статус-коды (403/404 для Gemini), лимиты токенов и корректность промптов.
- **Streamlit:** Помни о жизненном цикле (rerun). Проверяй `st.session_state` и логи в `streamlit.log`.
- **Python/Core:** Используй `logging` вместо `print`. Проверяй типы данных (type hints).

---

## Алгоритм действий
1. **Воспроизведение:** Найди минимальный набор действий/данных, вызывающий ошибку.
2. **Изоляция:** Отключи внешние зависимости (API), если это возможно, используя моки или тестовые скрипты (как `scripts/test_fix.py`).
3. **Гипотезы:**
   - "Ошибка 404 в Gemini вызвана использованием устаревшего эндпоинта v1beta."
   - "Streamlit падает из-за отсутствия ключа в session_state при первом запуске."
4. **Проверка:** Проверяй гипотезы по одной. Не меняй несколько участков кода одновременно.

## Инструменты
- `streamlit.log`: Основной источник ошибок UI.
- `docs/log.md`: Проверь список "Forbidden Models" перед сменой модели.
- `scripts/`: Используй существующие тестовые скрипты для проверки отдельных модулей `core/`.
