# Журнал работы (отладка Web UI DuckDNS, 01 марта 2026)

## Гипотезы (Hypotheses)
1. **Ошибка SSL (`ERR_CERT_COMMON_NAME_INVALID`)**: Nginx отдавал старый сертификат Beget вместо DuckDNS. Решение: принудительный выпуск `certbot --nginx -d aiconicagepro.duckdns.org` решил проблему.
2. **Ошибка "Cannot GET /chat" на порту 3000**: Был проброшен порт 3000, на котором вместо OpenClaw работал сторонний контейнер `gowa`. Решение: Поиск по логам и `ss -tulnp` подтвердили, что Control UI крутится на `18789`.
3. **Ошибка WebSocket (1008: control ui requires HTTPS or localhost)**: OpenClaw блокировал Nginx, считая его недоверенным прокси, и не видел HTTPS. Решение: добавлен заголовок `X-Forwarded-Proto https`, настроен жесткий редирект на HTTPS, а IP `127.0.0.1` добавлен в `gateway.trustedProxies` через `config set`.

## Успехи (Wins)
- Устранен конфликт портов Nginx (перенаправили трафик с ошибочного 3000 на верный 18789 для Web UI).
- Починен HTTPS для DuckDNS (зеленый замочек) через бесплатный Let's Encrypt сертификат.
- Сброшен забытый пароль от Nginx Basic Auth (`htpasswd`) на новый.
- Настроено защищенное WSS/HTTPS соединение для дашборда OpenClaw, преодолена встроенная блокировка (1008). Web UI успешно загружается.
- Все эти шаги задокументированы на русском языке (согласно глобальным правилам пользователя).
