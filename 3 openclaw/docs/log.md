# Журнал работы (отладка Web UI DuckDNS, 01 марта 2026)

## Гипотезы (Hypotheses)
1. **Ошибка SSL (`ERR_CERT_COMMON_NAME_INVALID`)**: Nginx отдавал старый сертификат Beget вместо DuckDNS. Решение: принудительный выпуск `certbot --nginx -d aiconicagepro.duckdns.org` решил проблему.
2. **Ошибка "Cannot GET /chat" на порту 3000**: Был проброшен порт 3000, на котором вместо OpenClaw работал сторонний контейнер `gowa`. Решение: Поиск по логам и `ss -tulnp` подтвердили, что Control UI крутится на `18789`.
3. **Ошибка WebSocket (1008: control ui requires HTTPS or localhost)**: OpenClaw блокировал Nginx, считая его недоверенным прокси, и не видел HTTPS. Решение: добавлен заголовок `X-Forwarded-Proto https`, настроен жесткий редирект на HTTPS, а IP `127.0.0.1` добавлен в `gateway.trustedProxies` через `config set`.

## Успехи (Wins)
- Устранен конфликт портов Nginx (перенаправили трафик с ошибочного 3000 на верный 18789 для Web UI).
- Починен HTTPS для DuckDNS (зеленый замочек) через бесплатный Let's Encrypt сертификат.
- Сброшен забытый пароль от Nginx Basic Auth (`htpasswd`) на новый.
- Настроено защищенное WSS/HTTPS соединение для дашборда OpenClaw, преодолена встроенная блокировка (1008). Web UI успешно загружается.
- Все эти шаги задокументированы на русском языке (согласно глобальным правилам пользователя).

## Архитектура БД (Redis и Postgres vs Supabase)
- **Почему Evolution API использует свой Postgres/Redis?**: Этот шлюз использует тяжеловесную библиотеку *Baileys* под капотом, которая генерирует гигабайты технических ключей сессии, кэширует статус сети, обрывы связи и служебные события WhatsApp. Если направлять этот поток прямо в Supabase (по сети), база мгновенно забьется техническим спамом, увеличатся задержки связи, и WhatsApp будет "отваливаться". Своя локальная база работает как кэш микро-сервиса.
- **Где же будет Supabase?**: Мы будем использовать Supabase (вместе с ИИ-агентом и фичей векторного поиска) на следующем слое "Воронки", в качестве **CRM**. 
- Вся техническая "грязь" остается в Evolution API. А в **Supabase** (при помощи вебхуков через N8N) мы отправляем уже *чистые диалоги, карточки Лидов и Сделок*. В Supabase будет лежать чистый контекст продаж, по которому Gemini будет искать смыслы с помощью векторного (Vector) поиска!

## Поиск шаблонов N8N: Два ИИ-пилота (Суфлер)
- Пользователь поставил задачу найти шаблоны для 2 пилотов: 1 анализирует, 2 подсказывает.
- Изучена база шаблонов MCP (`mcp_n8n-mcp_search_templates`). Выявлены похожие бизнес-схемы (RAG, автоответчики).
- Принято решение не применять автоответчик (чтобы не отпугнуть клиентов глупыми ответами), а использовать архитектуру "Human-in-the-loop" (Суфлер): Пилот 1 фильтрует webhook от Evolution, Пилот 2 пишет 3 черновика (драфта) и шлет их менеджеру в Telegram. Менеджер кликает "Отправить вариант 1", и N8N пушит готовый текст обратно в WhatsApp. План обновлен.
