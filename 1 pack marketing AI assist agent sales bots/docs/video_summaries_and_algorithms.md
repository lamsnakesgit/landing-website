# Саммари видео и пошаговые алгоритмы

## Видео 1: "Outputs lag relative to inputs" (iQFTolh1IAk)
**Суть:** Результаты (выводы/доходы) всегда отстают от вложений (усилий). Если вы надеетесь на быструю обратную связь, вы рискуете бросить прибыльное дело на полпути. Нужно постоянство.

**Пошаговый алгоритм для применения:**
1. **Сформулировать гипотезу:** Зафиксировать целевое действие (например, "отправлять 10 видео писем (Loom) в день").
2. **Определить срок "слепого" выполнения:** Выбрать период (минимум 30 дней), в течение которого действия выполняются строго каждый день.
3. **Заморозить оценку результатов:** В течение этого периода полностью игнорировать отсутствие лидов/откликов. Аналитика "отключена" в голове.
4. **Умножение ожиданий (Эвристика):** Умножить ожидаемое время получения результата на 2 или 3, чтобы дать себе реальные сроки.
5. **Анализ и итерация:** Только по истечении полного умноженного срока собрать данные и скорректировать подход.

## Видео 2: "Free AI Automation Agency Business Masterclass" (r0c7RhMFcww)
**Суть:** Создание прибыльного AI-агентства требует фокуса на классических бизнес-процессах (лидогенерация, созвоны, выполнение обязательств - 80%), а не только на технологиях (20%). В техническом плане главными навыками являются интеграция по API, работа с Webhooks и правильный структурированный AI-промптинг (JSON).

**Пошаговый алгоритм для применения:**
1. **Фокус на пайплайне продаж (Бизнес-база):**
   - Уделять основное время: Лидогенерация (Холодные email, Контент, Рефералы) -> Созвоны -> Предложение -> Выполнение (Интеграция ИИ).
2. **Освоение API (Application Programming Interface):**
   - Найти API нужного сервиса (например, Firecrawl для скрейпинга).
   - Изучить метод аутентификации (обычно `Authorization: Bearer <TOKEN>`).
   - Использовать библиотеку cURL из документации API, чтобы быстро перенести готовый запрос в блок `HTTP Request` в вашей системе (n8n/Make).
3. **Внедрение Webhook-интеграций (Отправка и Прием):**
   - Вместо ожидания готовых модулей, настроить кастомный Webhook-узел в n8n/Make.
   - Подключить URL этого Webhook к внешней CRM/задачнику (например, ClickUp "При создании задачи -> отправить запрос на Webhook").
4. **Структурированный AI-промптинг:**
   - Разделять роли в промптах: `System` (Кто такой ИИ, например: топ-копирайтер), `User` (Что нужно сделать, например: написать pitch), `Assistant` (Формат вывода).
   - Всегда просить AI возвращать результат в формате `JSON` с фиксированными ключами (например: `{"subject": "...", "pitch": "..."}`). Это позволяет надежно передавать эти данные в другие узлы рабочего процесса.
5. **Тестирование (Test-driven development):** Сперва создавать минимально рабочий процесс API-запроса, убедиться в его успешности, а уже затем надстраивать логику вокруг него.

## План внедрения алгоритмов в ассистентов (на n8n и OpenClaw)

### 1. Ассистент-автоматизатор на базе n8n
**Архитектурная роль:** n8n выступает "движком" для фоновых процессов (рассылки, парсинг, обогащение лидов, генерация предложений).

- **Шаг 1: Правильная системная архитектура (из Видео 2)**
  - Переключить запуск цепочек (trigger) на Webhooks. Использовать API-интеграции (HTTP Request) напрямую к целевым сервисам. Опираться на документацию API, а не на встроенные (порой обрезанные) узлы n8n.
- **Шаг 2: Внедрение структурированного ответа (JSON prompt)**
  - В узле запроса к LLM указать в System Prompt: "Верни ответ строго в валидном JSON-формате по такой структуре: ...".
  - На выходе из LLM добавить узел `Parse JSON` для гарантии обработки ответа и легкой передачи переменных (ключей) далее по воронке (в Telegram, Google Sheets, CRM).
- **Шаг 3: Автоматизация "Слепого" постоянства (из Видео 1)**
  - Настроить CRON-узел (Триггер по расписанию). Заставить n8n выполнять рутинные действия по генерации и отправке (например, 10 email в день) строго каждый день.
  - Ассистент должен продолжать рассылку независимо от "открытий писем", обеспечивая системный input.

### 2. Ассистент-коммуникатор на базе OpenClaw
**Архитектурная роль:** OpenClaw используется как диалоговый интеллект с доступом к инструментам (Tools) и базе знаний для общения.

- **Шаг 1: Настройка ролей (System Prompts - Видео 2)**
  - В настройках агента OpenClaw жестко задать системную роль: "Ты ведущий системный архитектор и специалист по продажам AI-агентства...". Эта роль никогда не должна сбиваться.
- **Шаг 2: Подключение Webhook/API как Tools (Инструменты агента)**
  - Реализовать вызов кастомных инструментов: настроить в OpenClaw инструмент, который отправляет запрос на Webhook n8n. Например: Агент просит n8n собрать данные о сайте клиента, n8n делает скрейпинг, возвращает JSON в OpenClaw, а OpenClaw переводит его в читаемый текст для пользователя.
- **Шаг 3: Мониторинг "Отставания результатов" (Outputs lag - Видео 1)**
  - Добавить агенту инструмент для ведения журнала действий (Daily Log). Запрашивать у него еженедельный отчет по количеству выполненных inputs (лидов сгенерировано, писем отправлено), чтобы отслеживать стабильность вложений в течение месяца без оглядки на быстрые финансовые возвраты.
