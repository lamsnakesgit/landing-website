# Стратегия WhatsApp Рассылки и Отложенных Задач в n8n

Этот документ описывает архитектуру и требования к системе автоматизации WhatsApp и управления будущими задачами.

## 1. WhatsApp Сообщества и Чаты
### Требования:
- Поддержка иерархии "Сообщество -> Чаты".
- Возможность массового включения/исключения чатов конкретного сообщества.
- Использование `parentJid` в Evolution API для определения принадлежности к сообществу.

### Реализация:
- **Синхронизация**: Ежечасный workflow в n8n скачивает список всех чатов.
- **Data Tables**: Чаты сохраняются в таблицу n8n с полями `jid`, `name`, `parent_jid`, `type`, `link`.
- **Проверка дубликатов**: Новые чаты добавляются только если их JID отсутствует в таблице.
- **Уведомления**: При обнаружении новых чатов отправляется отчет в Telegram (Общее кол-во + список новых с ссылками).

## 2. Система отложенных задач (Отложенные напоминания)
### Требования:
- Создание напоминаний или действий на любую дату в будущем.
- Устойчивость к перезагрузкам n8n.
- Регулярные и разовые действия.

### Реализация (Supabase + n8n):
- **Хранилище**: Таблица в Supabase (Free Tier: 500MB БД, 5GB трафик — достаточно для текста).
- **Безопасность**: Row Level Security (RLS) + Service Role Key.
- **Диспетчер**: Workflow в n8n каждые 5 минут проверяет БД на наличие задач, время исполнения которых наступило (`execute_at <= NOW()`).
