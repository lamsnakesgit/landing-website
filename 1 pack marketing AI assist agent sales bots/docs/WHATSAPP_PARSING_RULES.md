# 🕵️ Правила парсинга контактов и LIDs в WhatsApp (2024-2026)

Этот документ описывает суровые реалии сбора баз номеров из WhatsApp, объясняет, откуда берется "мусор" (`@lid`) в базах и почему парсинг групп больше не является панацеей для лидогенерации.

---

## 1. Что такое LIDs (Lightweight Identifiers)?
При парсинге групп через Evolution API (или любую другую библиотеку на базе Baileys) вы можете столкнуться с тем, что вместо привычных номеров форматов `79991234567@s.whatsapp.net` API возвращает вам "дичь" — короткие ID с окончанием `@lid` (например: `123123123:2@lid`).

**Почему это происходит:**
Это результат внедрения новой системы приватности от Meta (WhatsApp). Те пользователи, чьи номера вы не видите, скрыли свои номера настройками приватности, либо находятся в Сообществе (Community). Для защиты от спама WhatsApp заменяет их реальный номер телефона криптографическим алиасом (LID).

**Важное правило:** Из LID невозможно восстановить реальный номер телефона пользователя.

---

## 2. Архитектура приватности WhatsApp

### А. Сообщества (Communities / Группа объявлений)
*   **Сбор номеров обычным участником:** **НЕВОЗМОЖЕН**. Номера всех участников (кроме админов) скрыты на уровне архитектуры WhatsApp по умолчанию. Нет никаких галочек или настроек, телефон просто не получает эти данные с сервера. Вы парсите только пустоту или LIDs.
*   **Сбор номеров Админом:** Возможен. Если ваш аккаунт (привязанный к Evolution API) является **администратором** сообщества, сервер доверяет вам и отдает реальные мобильные номера всех участников (`@s.whatsapp.net`).

### Б. Обычные группы и подгруппы
*   Работает **двойной контроль**:
    1.  **Настройка участника:** Если человек в личных настройках скрыл номер от тех, кого нет в телефонной книге — он превратится в LID при парсинге.
    2.  **Настройка админа:** Админ может поставить галку "Скрыть участников" для всей группы. Тогда никто, кроме админа, даже не увидит список людей.
*   **Но:** Если группа публичная, "старого" типа и приватность не закручена — номера собираются без проблем. К сожалению, таких групп с развитием Сообществ становится всё меньше.

## 3. "Хаки" Baileys: Можно ли слать сообщения на LIDs?

Технически — `Да`. Практически — `Забудьте об этом для рассылок`.

**Как это работает:**
Движок Baileys (ядро Evolution API) эмулирует WhatsApp Web. Он умеет резолвить связи. Если сервер WhatsApp прислал вам `LID`, значит внутри экосистемы этот `LID` всё равно жестко привязан к реальному номеру. 
Технически вы можете передать `123456:12@lid` в поле `number` при отправке сообщения в Evolution API, и сообщение может быть доставлено (иногда требуется трюк с отправкой VCard).

**Почему это не использовать для лидогенерации:**
1.  **Мгновенный БАН:** Отправка сообщения пользователю, который явно скрыл свой номер и с которым у вас нет предварительного чата — это краснейший флаг для анти-спам системы Meta. Аккаунт будет заблокирован почти моментально.
2.  **Нулевая сегментация:** Вы шлете сообщение в пустоту. Вы не знаете ни кода страны LIDа, ни кто это такой. 
3.  **Ненадежность:** Доставка сообщений на LIDs (без наличия истории переписки) часто блокируется самим сервером и сыпет ошибками в API.

## 4. Как чистить базу при парсинге (n8n Solution)

Если вы всё-таки парсите открытые чаты, всегда пропускайте данные через фильтр в n8n Code Node, чтобы отсечь мусор и оставить только "чистые" номера:

```javascript
// Код для Node.js в n8n (нода Code)
const participants = $json.participants || [];

// Оставляем ТОЛЬКО тех, у кого нормальный номер (@s.whatsapp.net)
const realNumbers = participants.filter(user => 
  user.id.includes('@s.whatsapp.net') && 
  !user.id.includes('@lid')
);

// Возвращаем чистый список
return realNumbers.map(user => {
  return {
    json: {
      phone: user.id.split('@')[0], // Отсекаем хвост до @
      id: user.id,
      admin: user.admin
    }
  }
});
```

## Вывод / Стратегия
Парсинг защищенных WA-групп — это пустая трата времени для B2B продаж. Гораздо эффективнее и безопаснее получать номера из официальных или открытых B2B источников:
*   **2ГИС** (поиск компаний)
*   **HH.ru** (контакты HR/РОП)
*   **Госзакупки** (контакты ИП/ТОО)
*   **Adata.kz** (реестр)

А WhatsApp использовать не как источник поиска лидов, а как **исключительный канал прогрева и продажи** для уже найденных, квалифицированных холодных баз.
