# Анализ архитектуры: Telegram AI Assistant vs WhatsApp Summary Agent

Этот файл объясняет, почему два воркфлоу выглядят совершенно по-разному, какова логика каждого из них, и как настроить WhatsApp-версию для работы.

---

## 1. В чем главная разница и почему они так отличаются визуально?

**Telegram AI-Ассистент (кодовое имя Aikicka: ~40 нод)**
Это огромный "мультимодальный комбайн" на базе Telegram. 
Что он делает:
- Общается текстом (имеет память через LangChain).
- Принимает голосовые сообщения и кружки (Видео), переводит их в текст.
- Сам отвечает **голосом** (через упавшую TTS-ноду Edge).
- Генерирует **фотографии** по текстовым промптам (через Seedance / Stable Diffusion).
- Проверяет **базу данных Supabase / Google Sheets** (кто сколько запросов потратил, кому закрыть доступ).
- Выдает форматированные HTML-сообщения.

**WhatsApp Summary Agent (который я создал сейчас: всего 7 нод)**
Это "хирургический скальпель" с одной-единственной задачей. Ты изначально просил *"чтоб он в вотсап чатах тоже работал, типа он делает саммари в чате"*. 
Поэтому я вырезал из него абсолютно весь лишний код (генерацию картинок, озвучку голосом, проверку баланса базы данных). 
Что он делает:
- Слушает групповой чат (или личку) в WhatsApp.
- Ловит **только** голосовые сообщения. Весь текст, фотки и видео он просто игнорирует (чтобы не спамить в ответ на каждую реплику коллег в группе).
- Как только ловит голос — транскрибирует -> скармливает ИИ -> отправляет красивую текстовую выжимку обратно в группу, тегая автора.

> **Вывод:** Если тебе нужен в WhatsApp *такой же полноценный ассистент-комбайн* (чтобы он отвечал картинками, общался текстом и проверял баланс юзера по БД), нам нужно будет создать новый воркфлоу, перенеся всю ту логику на ноды Evolution API. Шаблон Саммари-бота был сделан именно для автоматизации рутины (послушать долгое аудио -> отдать короткий текст).

## 2. Разница в инфраструктуре (Telegram vs Evolution API)

| Фича | Telegram (Оригинал) | WhatsApp (Evolution API) |
| :--- | :--- | :--- |
| **Прием сообщений** | Специальная нода `Telegram Trigger`, которая сама скачивает файлы. | Обычная нода `Webhook`. Аудиофайл приходит **в виде гигантской строки Base64** внутри JSON. Поэтому в WhatsApp-шаблоне стоит нода `Code`, чтобы склеить этот Base64 обратно в нормальный файл. |
| **Отправка файлов** | Нода `Telegram (Send Photo/Audio)` принимает обычные файлы n8n. | В Evolution API файлы тоже отправляются как строка Base64 (через `mediaMessage`), что требует дополнительной подготовки в коде. |
| **Форматирование** | Используется `HTML` (`<b>`, `<i>`). | WhatsApp понимает **только Markdown** (`*жирный*`, `_курсив_`). Если отправить туда HTML, он выведет теги текстом. Именно поэтому в Саммари-боте я прописал ИИ жесткий промпт: "Использовать только символы WhatsApp". |
| **Понимание "В какой чат отвечать?"** | Переменная `{{$json.message.chat.id}}`. | Переменная `instance` (от какого твоего номера) + `remoteJid` (кому писать) + `participant` (кого тегнуть в группе). |

## 3. Инструкция: Как настраивать текущий WhatsApp Summary Agent

Чтобы запустить этот минималистичный шаблон, нужно сделать 3 простых шага:

### Шаг 1: Настройка получения данных (Webhook)
1. Открой первую ноду `Evolution Webhook` в n8n (в режиме Production). 
2. Скопируй её URL-адрес (`https://твой-n8n/webhook/whatsapp-summary`).
3. Зайди в настройки твоего инстанса Evolution API (через Postman или UI-панель).
4. В разделе **Webhooks** включи отправку событий (`messages.upsert`).
5. Вставь скопированный URL.
6. **КРИТИЧНО:** Включи в настройках вебхука `base64: true` (отправлять медиафайлы в base64). Если это не включить, n8n не сможет получить аудиофайл.

### Шаг 2: Настройка ИИ (Транскрипция и Мозг)
1. Нода `Whisper Transcribe`: уже настроена на твой сервер Railway (`openai-whisper-asr-...`). Пока он жив — трогать не нужно.
2. Нода `Google Gemini`: зайди в нее и выбери (или создай) `Credential` типа **Google Gemini(PaLM) Api account**. Вставь туда свой бесплатный ключ `AIzaSy...`.

### Шаг 3: Настройка отправки обратно (HTTP Request)
1. Открой последнюю ноду `Send back to WhatsApp`.
2. В параметрах **Send Headers** найди поле `apikey`.
3. Замени значение `YOUR_EVOLUTION_GLOBAL_API_KEY` на твой глобальный пароль от сервера Evolution API (тот, что ты используешь для авторизации в Postman).
4. В самом URL прописан айпи-адрес сервера (`http://151.241.100.226:8080`). Убедись, что Evolution API крутится именно там и порт открыт для n8n.

Всё! Сохрани воркфлоу и переключи тумблер **Active** в правом верхнем углу. Теперь, кто бы ни отправил тебе (или в группу) голосовое сообщение, бот пришлет текстовую выжимку.
