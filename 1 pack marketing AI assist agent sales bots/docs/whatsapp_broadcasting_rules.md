# Правила и архитектура WhatsApp-рассылок (Evolution API)

Этот документ описывает ключевые принципы работы с серым (неофициальным) API WhatsApp для массовых рассылок и лидогенерации.

## 1. Время жизни сессии
*   **Длительность:** Теоретически сессия Evolution API бессрочна. Она может жить месяцами и годами.
*   **Разрыв сессии:** Сессия разрывается только в трех случаях:
    1. Вы вручную нажали "Выйти со всех устройств" в самом приложении WhatsApp на телефоне.
    2. Телефон, на котором работает WhatsApp (основной), не был подключен к интернету более 14 дней.
    3. WhatsApp принудительно разлогинил сессию из-за подозрений в спаме (или забанил номер).
*   **Рекомендация:** Держите телефон-донор заряженным и подключенным к Wi-Fi/интернету.

## 2. Количество номеров (Инстансов)
*   В Evolution API можно создать **неограниченное число инстансов (номеров телефона)**. Здравствуйте, коллега! Пишу вам от лица компании Aiconic Agency...
*   Каждый номер подключается через отдельный QR-код как отдельное "Связанное устройство".
*   Ограничение упирается только в ресурсы вашего VPS-сервера. Один сервер за $5-10 легко тянет 10-20 одновременно работающих номеров.

## 3. Ротация номеров (На будущее)
Для масштабных рассылок (когда нужно слать по 1000+ сообщений в день) с одного номера писать нельзя — это 100% бан. В этом случае используется **Ротация (Round-Robin)**:
1. Вы подключаете к Evolution API 10 разных номеров (SIM-карт).
2. В n8n создается механизм, который при отправке рассылки берет массив из этих 10 номеров.
3. n8n отправляет сообщения по очереди: 1-е сообщение с первого номера, 2-е со второго, 3-е с третьего... и так далее по кругу.
4. В итоге каждый из 10 номеров отправляет всего по 100 сообщений в день, что выглядит для WhatsApp как нормальная активность живого человека.

## 4. Правила Анти-Бана (Повторение)
*   **Спинтакс:** Обязательная рандомизация текста `[Привет|Здравствуйте|Добрый день]` в n8n (`Spintax`).
*   **Случайные задержки:** Паузы между отправками сообщений (от 2 до 10 минут) через ноду `Wait` в n8n.
*   **Прогрев:** Новый номер должен "пожить". Сначала пишите контактам из адресной книги, чтобы они отвечали, вступайте в группы. Не начинайте холодную рассылку в первый же день.
*   **Без ссылок в первом сообщении:** Ссылки шлем только после того, как клиент хоть что-то ответил (что автоматически снимает его с фильтра спама).
*   **Наличие аватарок и "О себе":** Номера должны выглядеть как живые люди, а не пустые боты.

## 5. Как хранить переписки (CRM и Supabase)
Сама по себе отправка сообщений (как в базовом воркфлоу) недостаточна. Нам важен **результат** и **контекст**.
Идеальным решением является использование базы данных **Supabase** (как более мощной альтернативы Google Sheets), так как она позволяет удобно хранить массивы (JSON) переписок в одной ячейке базы данных.

**Как организовать таблицу лидов (Смотри `assets/leads_crm_sample.csv`):**
1.  `phone` (Уникальный ID клиента)
2.  `status` (New, Sent, Replied, Interested, Closed, Bounced)
3.  `conversation_history` (Здесь хранится весь массив сообщений в JSON `[{"role": "user", "text": "Да, интересно"}, {"role": "bot", "text": "Тогда вот презентация..."}]`).
4.  `last_reply_received_at` (Дата последнего ответа — нужна для воркфлоу отправки дожимов/follow-ups).

| id | phone | name | company | status | last_message_sent_at | last_reply_received_at | conversation_history | assigned_agent | lead_score |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | +1234567890 | Иван | LogisticsPro LLC | New | | | | | |
| 2 | +0987654321 | Анна | FastTrack Cargo | Sent | 2026-03-06 10:00:00 | | | AI_Sales | |
| 3 | +1122334455 | Петр | Global Ship | Replied| 2026-03-05 15:00:00 | 2026-03-05 15:30:00 | `[{"role": "bot", "content": "Здравствуйте!"}]` | AI_Sales | 80 |
| 4 | +9988776655 | Дмитрий| TransLog | Sent | 2026-03-04 09:00:00 | | | AI_Sales | 10 |

Когда человек отвечает в WhatsApp, срабатывает Webhook в n8n, n8n ищет его строку в Supabase по `phone`, добавляет новое сообщение в массив `conversation_history`, меняет `status` на `Replied`, и передает весь этот массив в LLM (OpenAI) как контекст для следующего ответа.

## 6. Потребление ресурсов Evolution API
*   **Сколько "весит" 1 номер?** 
*   Evolution API написан на Node.js и использует библиотеку Baileys (которая общается с WhatsApp по WebSockets).
*   Сам движок в простое потребляет минимум ресурсов. **Один инстанс (подключенный номер)** в простое забирает примерно **50-100 МБ оперативной памяти (RAM)**.
*   Если у вас дешевый VPS (1 GB RAM, 1 CPU), он легко потянет **5-10 инстансов** одновременно (если они не перекидывают гигабайты видео каждую секунду). Для 20-30 инстансов лучше взять сервер на 2-4 GB RAM.

## 7. Риски виртуальных номеров (SMS-Activate и т.д.)
**Вопрос:** *Разве другой человек не может перехватить WhatsApp, купленный на SMS-Activate за копейки?*
**Ответ:** Да, это **самая большая уязвимость** этого метода!
Дешевые номера на 20 минут выдаются вам только для получения одной SMS. Через время этот номер возвращается в общую базу сервиса. Если кто-то другой решит купить номер для WhatsApp и ему случайно выпадет тот же номер — он запросит SMS, введет код, и **ваша сессия в EvolutionAPI мгновенно оборвется**, аккаунт будет угнан.

**Как арбитражники решают эту проблему:**
1.  **Двухфакторная аутентификация (2FA / PIN-код):** Как только арбитражник регает WhatsApp на вирт. номер, он МГНОВЕННО ставит PIN-код в настройках безопасности WhatsApp. Даже если номер перекупят и получат SMS, войти в аккаунт без вашего PIN-кода будет невозможно. (Но WhatsApp может сбросить PIN через 7 дней отсутствия активности).
2.  **Аренда номеров:** В сервисах типа SMS-Activate есть вкладка "Аренда". Вы берете номер в аренду на месяц (это стоит $2-5). В течение месяца номер принадлежит только вам, никто другой SMS на него не получит.
3.  **Физические SIM-банки:** Серьезные команды покупают реальные SIM-карты тысячами (оптом по 50-100 руб/шт) на рынках/переходах, вставляют их в GSM-шлюзы и регистрируют надежно.

## 8. Авто-прогрев номеров (Farm/Warmup)
Смысл авто-прогрева в том, чтобы заставить новые номера переписываться друг с другом, имитируя бурную жизнь реальных людей, чтобы повысить "траст" (доверие) со стороны анти-спам алгоритмов Meta. 
Вот как выглядит флоу:

```mermaid
graph TD;
    A[Новый номер + Вася] -->|n8n запускает диалог| B[Новый номер + Петя]
    B -->|Через 5 мин Петя отвечает| A
    C[Новый номер + Маша] -->|Пишет в группу| D[Группа "Прогрев Ботов"]
    B -->|Петя пишет Маше| C
    style A fill:#e1f5fe,stroke:#03a9f4,stroke-width:2px;
    style B fill:#ffebee,stroke:#f44336,stroke-width:2px;
    style C fill:#f3e5f5,stroke:#9c27b0,stroke-width:2px;
    style D fill:#e8f5e9,stroke:#4caf50,stroke-width:2px;
```

**Workflow Авто-прогрева в n8n:**
1.  В Supabase есть база из 10 ваших "купленных" бот-инстансов.
2.  Каждому боту присваивается "характер" или промпт (например, "Ты строитель, любишь короткие сообщения и стикеры").
3.  Раз в 15 минут n8n берет случайного бота (А) и заставляет его написать случайному боту (Б).
4.  Эволюшн бота Б принимает Webhook (входящее сообщение).
5.  ChatGPT бота Б читает сообщение от бота А, пишет органичный ответ, и отправляет обратно.
6.  Они могут обмениваться фотками из интернета, голосовыми (сгенерированными нейронкой) и карточками контактов.
Так номера "греются" 5-7 дней, прежде чем вы начнете слать с них рассылки по реальным холодным базам.

## 9. Лимиты отправки (Anti-Ban стратегия)
В отличие от официального Cloud API, Evolution API не имеет жестких счетчиков, но WhatsApp жестко банит за подозрительную активность и жалобы на "Спам".

**Безопасный старт (Первые 5 дней на новых номерах):**
*   **День 1:** 5-10 сообщений знакомым (тем, кто 100% ответит).
*   **День 2:** 15-20 сообщений.
*   **День 5:** до 50 сообщений.

**Боевые лимиты прогретого номера:**
*   **Холодная база (высокий риск жалоб):** Максимум **40-60 сообщений в день** с одного инстанса. Обязательна задержка (`Random Delay`) в n8n от 2 до 5 минут между отправками.
*   **Теплая база (старые клиенты):** 200-300 сообщений в день.

**Золотое правило:** Бан зависит от процента нажатий кнопки "Спам". Если из 100 холодных лидов 5 нажмут "Спам" — номер улетает в блок. Разделяйте риски: используйте 5-10 номеров (Round-Robin) для рассылки 500 сообщений в день.

## 10. По каким базам слать (B2B Логистика и Маркетинг)
Секрет успеха — узкие, максимально целевые базы, а не слепой парсинг всех подряд.

**Для Логистической компании (Поиск импортеров/экспортеров):**
1. **Участники B2B выставок:** Парсите сайты прошедших и будущих выставок (MosBuild, Продэкспо, выставки Китая). Отправляйте сообщение: *"Иван, видели вас в списках участников MosBuild. Везем сборный груз из Китая/Европы, есть свободное место. Актуально?"*
2. **Таможенные базы ВЭД:** Покупаются актуальные базы таможенных деклараций. Вы точно знаете, что компания ООО "Вектор" месяц назад ввезла станки из Китая. Заходите с козырей.
3. **2GIS / Яндекс.Карты (Узкие ниши):** Парсинг сетей магазинов одежды, дистрибьюторов стройматериалов, автозапчастей в связке с парсингом контактов ЛПР (руководителей) через GetContact или аналоги.
4. **Участники профильных Telegram-чатов:** Парсинг чатов типа "Бизнес с Китаем", "Поиск грузов", "Оптовики РФ".

**Для Aiconic Agency (Поиск B2B клиентов на ИИ-ботов и автоматизацию):**
1. **Онлайн-школы и Инфобизнес:** Парсите GetCourse, каталоги онлайн-курсов, участников инфоконференций. Им безумно нужны ИИ-продажники.
2. **Агентства недвижимости и Застройщики:** Сайты недвижки, Циан. Бот для квалификации лидов в недвижимости экономит миллионы на зарплатах Call-центрам. 
3. **Медицинские клиники и Стоматологии:** Парсинг 2GIS. ИИ-бот для круглосуточной записи на прием, ответов по стоимости и напоминаний.
4. **E-commerce (Интернет-магазины):** Сайты на Tilda, Shopify, InSales. Им нужен бот техподдержки и генерации контента.

## 11. Парсинг WhatsApp-групп (Технические ограничения)
Частый запрос от клиентов (например, в логистике): *"сделайте парсер участников чужих групп"*.
Evolution API действительно умеет мгновенно выгружать все телефоны участников группы (через эндпоинты `/group/findGroupInfos` или `fetchGroupParticipants`). Но есть **жесткие ограничения ядра WhatsApp**, о которых нужно знать:

1. **Вы ОБЯЗАНЫ состоять в группе.** Парсер работает только изнутри. Если это закрытая VIP-группа оптовиков, ваш номер-бот должен подать заявку, и админ должен ее одобрить. Спарсить группу "снаружи" невозможно (это защищено шифрованием WhatsApp).
2. **Вышедшие и удаленные:** Если вас удалили из группы или вы вышли сами — доступ к серверной информации о группе мгновенно сгорает. Спарсить участников группы, в которой вас больше нет, **технически невозможно**.
3. **Скрытые списки (Communities):** Если это не обычная группа, а WhatsApp Community (Сообщество), и администратор скрыл список участников от обычных людей, то парсер ничего не увидит даже находясь внутри. Выгрузить скрытых участников может только админ группы.
4. **Безопасная тактика:** Для логистики идеальный парсинг выглядит так: зайти с 5-6 свежих номеров в 100 открытых целевых чатов (по ссылкам-инвайтам из Telegram или сайтов), подождать сутки, запустить n8n воркфлоу, который выгрузит все номера участников этих групп в таблицу, а затем медленно писать им с "чистых" номеров по холодной рассылке, ссылаясь на то, что *"мы состоим в одной группе по логистике"*.
