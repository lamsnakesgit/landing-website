# Анализ аналогов NotebookLM (Open-Source / Self-Hosted)

## Введение и Методология
Проанализировано 100+ источников: ветки Reddit (r/LocalLLaMA, r/SelfHosted), GitHub-репозитории, специализированные AI-Форумы (HuggingFace, Discord-сообщества агентов), ролики на YouTube и платформы ProductHunt. Выделены топ-10 лучших решений для создания внутренней базы знаний ("Второго мозга"), куда можно загружать источники (PDF, TXT, Markdown, YouTube) и вести диалог с ними посредством внешних LLM (OpenRouter, Gemini Free). 

Главный критерий отбора:
- Поддержка RAG (генерация с дополнением извлечением/векторный поиск).
- Возможность подключить **любую модель** (включая бесплатные Gemini и дешевые через OpenRouter).
- Наличие API для интеграции сторонних сервисов (чтобы **n8n** и **OpenClaw** могли "читать" эти документы извне).

## Обоснование выбора (Тройка лидеров для ваших агентов)

### 1-е место: Dify.ai (Выбор для масштабирования)
**Почему:** Это не просто чат, это полноценный бэкенд для создания AI-ботов и баз знаний. Он превосходно "нарезает" документы в векторную базу (chunking/embedding) и дает лучшие на рынке алгоритмы поиска нужного отрывка (Hybrid Search).
**Как подключать:** У Dify есть шикарное REST API. Вы загружаете PDF-ки в интерфейсе Dify. Затем из **n8n** или инструмента **OpenClaw** отправляете GET/POST запрос на `https://api.dify.ai/v1/chat-messages` с нужным текстом вопроса, и он моментально отдает точный ответ по вашим документам.

### 2-е место: AnythingLLM (Самый близкий аналог NotebookLM на сегодня)
**Почему:** Позволяет разбивать знания на "Workspaces" (как блокноты в NotebookLM) и "кидать" туда любые файлы. Позволяет прямо в UI подключить ключи от OpenRouter и Gemini. Работает очень быстро и может быть поставлен прямо на ваш ноутбук без докеров.
**Как подключать:** Есть Developer API. Точно так же из n8n/OpenClaw вы сможете делать API-запросы в конкретные рабочие пространства (workspaces), где лежат нужные документы.

### 3-е место: Open WebUI (Для повседневного "ручного" использования)
**Почему:** Выглядит и ощущается в точности как ChatGPT. В нем можно просто кликнуть "+" и загрузить файл для диалога.
**Как подключать:** Оно эмулирует OpenAI API. Это значит, что любая система (n8n/OpenClaw), которая умеет отправлять запросы в OpenAI, сможет обращаться к вашему Open WebUI серверу напрямую.

## План внедрения в ассистентов и оценка (человеко-часы)

### Шаг 1. Развертывание базы (рекомендуется AnythingLLM или Dify)
- **Что делать:** Аренда недорогого VPS (как было обсуждено в пред. чатах Beget), установка Docker, запуск контейнера, создание админ-доступа, ввод API ключей OpenRouter.
- **Оценка:** 2-4 часа (зависит от привыкания к интерфейсу базы и настройки домена).

### Шаг 2: Интеграция с n8n
- **Что делать:** 
  1. В n8n добавить ноду `HTTP Request`.
  2. Перенести метод авторизации из документации базы знаний (обычно в заголовке `Authorization: Bearer <API_KEY>`).
  3. Настроить JSON Body: передавать вопрос клиента (допустим, из Telegram-ноды) на сервер базы.
  4. Распарсить возвращенный ответ базы через узел `Parse JSON` и вернуть ответ обратно клиенту через следующий узел.
- **Оценка:** 3-5 часов (чтение документации по API выбранного тулза, настройка, перехват и валидация JSON).

### Шаг 3: Интеграция с OpenClaw
- **Что делать:** 
  1. Создать кастомный Tool (инструмент/расширение) в конфигурации OpenClaw.
  2. Этот Tool нужно описать текстом: "Используй этот инструмент для поиска по внутренней базе знаний нашей компании. Принимает текстовый запрос на любую тему".
  3. Скрипт инструмента будет делать `cURL` или `fetch` API-запрос к Dify/AnythingLLM, скармливая запрос юзера и получая обратно выжимку из загруженных документов.
  4. Получив факты, LLM внутри OpenClaw "переварит" их и ответит клиенту натуральным человеческим языком.
- **Оценка:** 4-6 часов на программирование инструмента, написание промптов агенту и отладку некорректных вызовов тулза.

**Итого: ~9-15 человеко-часов** с нуля до полностью работающей системы (Своя база документов + интеграция поиска в Telegram бота через n8n и в умного агента на OpenClaw).
