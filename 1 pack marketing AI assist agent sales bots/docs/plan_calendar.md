# План: AI-Ассистент для Календаря и Нетворкинга

## 1. Ежедневная сводка праздников
**Цель:** Каждое утро получать от бота сообщение с актуальными праздниками для генерации инфоповодов.
**Состав сводки:**
- Государственные и национальные праздники (КЗ, РФ, СНГ, Европа, Мир).
- Профессиональные праздники.
- Необычные/забавные международные дни.
**Частота:** 1 раз в день (утром, например, в 08:00).
**Реализация (n8n):**
- Источник данных: Google Календарь (подписки на Holiday календари) или внешнее API праздников (например, AbstractAPI, Calendarific).
- LLM обрабатывает сырые данные и формирует выжимку.
- Отправка в Telegram через Telegram Node.

## 2. Система нетворкинга (Кейт Феррацци / Дерек Сиверс)
**Цель:** Поддержка "слабых" и "сильных" связей, регулярные касания с окружением.
**Механика:**
- Каждому контакту (в CRM или Notion/Airtable/Supabase) присваивается **Группа приоритетности** и **Рейтинг влияния**:
  - *Группа A (Высокий приоритет, сильное влияние)*: касание каждые несколько дней / 1-2 раза в неделю.
  - *Группа B (Средний приоритет)*: касание раз в месяц.
  - *Группа C (Слабые связи, широкий круг)*: касание раз в квартал / раз в год.
**Работа бота:**
- Ежедневный запрос к базе контактов: "С кем сегодня нужно связаться по графику?".
- Бот выдает список из 3-5 человек на сегодня.
- **Инфоповоды для касания:** Бот автоматически предлагает причину для сообщения:
  - Связка с сегодняшним праздником (из пункта 1).
  - Связка с началом/концом месяца, квартала.
  - День рождения, годовщина или профессиональный праздник контакта.
  - Предложение: "Отправь эту полезную статью", "Предложи помощь в его сфере", "Просто спроси как дела после прошлого разговора".
  - Бот может сразу сгенерировать драфт (черновик) сообщения, который можно скопировать и отправить.

## 3. Обновление промптов и форматирования сообщений (Telegram)
**Проблема:** Стандартные Markdown-звездочки (`**жирный**`) часто парсятся некорректно в Telegram или выглядят "грязно" в виде сырого текста.
**Решение:**
- В настройках узла Telegram в n8n обязательно выбрать `parse_mode: HTML`.
- В системном промпте LLM указать жесткое правило: **"ЗАПРЕЩЕНО использовать Markdown (`**`, `*`, `#`). Используй только HTML теги `<b>жирный</b>`, `<i>курсив</i>`, `<u>подчеркнутый</u>` для выделения текста. Списки оформляй с помощью эмодзи или дефисов."**
- Использовать эмодзи для визуального разделения блоков (📅, 🎉, 🤝, 💡).
- Добавить абзацы для "воздуха" между блоком календаря и блоком нетворкинга.

## 4. Архитектура рабочего процесса (n8n Workflow)
1. **Trigger:** ScheduleTrigger (каждый день в 07:00 или 08:00).
2. **Data Gathering (Сбор данных):**
   - Узел *Google Calendar* (перебор календарей с праздниками) или *HTTP Request* (к API праздников).
   - Узел интеграции с БД (Postgres/Supabase/Notion) для извлечения контактов, у которых наступила дата касания.
3. **AI Processing / LLM Chain:**
   - Первичный промпт: Анализ праздников и формирование короткой структурированной выжимки.
   - Вторичный промпт: Анализ людей на сегодня -> Подбор инфоповода для каждого -> Генерация черновика сообщения (драфта).
   - Финальный промпт (или системный для агента): Свести всё в единый HTML-текст для Telegram.
4. **Output:** 
   - Узел *Telegram (Send Message)* с передачей HTML разметки.
