# Google Apps Script — Лимиты, Возможности, FAQ

## Лимит «6 минут на запуск» — что это значит?

Каждый раз когда ты нажимаешь **▶️ Рассылка** или срабатывает таймер — Google запускает твой скрипт. У него есть **6 минут** чтобы закончить работу. Если за 6 минут не успел — Google убивает процесс.

### Пример:
- Пауза между сообщениями = 10 сек
- За 6 минут (360 сек) можно отправить ~**30 сообщений**
- Если в базе 200 номеров → скрипт отправит 30 и Google его остановит

### Решение: батчи + таймер
Именно поэтому в v4 есть `BATCH_SIZE = 10`. Таймер каждый час запускает скрипт, он отправляет 10 штук за ~2 мин и завершается. Через час — ещё 10.

```
Час 1:  ▶️ 10 сообщений (2 мин) → ✅
Час 2:  ▶️ 10 сообщений (2 мин) → ✅
Час 3:  ▶️ 10 сообщений (2 мин) → ✅
...
За 24 часа = 240 сообщений, батч повторяется, пока не закончатся строки "отправить"
```

Скрипт НЕ нужно запускать второй раз вручную — **таймер делает это сам**.

### Google Workspace (платный) = 30 минут
Если у тебя платный Google Workspace аккаунт → лимит 30 мин вместо 6. Это ~150 сообщений за один запуск. Но батчи всё равно безопаснее для WhatsApp.

## Работает ли без компа?

**ДА!** Триггеры (таймеры) работают на серверах Google. Можешь:
- ✅ Выключить компьютер
- ✅ Закрыть браузер
- ✅ Уехать в отпуск
- ✅ Спать

Таймер будет слать батчи и проверять ответы автономно.

**НО:** `onOpen()` (меню) появляется только когда кто-то открывает таблицу. Таймерам это не нужно.

---

## Может ли Apps Script управлять браузером?

**НЕТ. ❌** Apps Script — это серверный код на Google. У него нет браузера, нет экрана, нет мыши.

Он может:
- ✅ Делать HTTP-запросы (к API)
- ✅ Работать с Google Sheets, Docs, Drive, Gmail
- ✅ Отправлять запросы к любому API
- ❌ Открывать сайты
- ❌ Кликать по кнопкам
- ❌ Заполнять формы на сайтах

### Для Threads авто-комментинга — варианты:

| Инструмент | Что умеет | Где работает |
|:---|:---|:---|
| **Playwright/Puppeteer** (Node.js) | Полное управление браузером | ВПС (24/7) |
| **Selenium** (Python) | То же самое | ВПС |
| **Chrome Extension** | Работает в твоём браузере | Только пока комп включён |
| **Threads API** (официальный) | Постинг через API, без браузера | Где угодно |
| **n8n + HTTP Request** | Через API, если есть | ВПС |

### Рекомендация для Threads:
1. **Если есть API** — лучше через API (Apps Script или n8n сможет)
2. **Если нет API** — Node.js + Playwright на ВПС (как твой WA sender.js)
3. **Быстро попробовать** — Chrome Extension (но нужен открытый комп)

По сути для Threads лучше сделать **отдельный Playwright-скрипт** на ВПС, похожий на тот что мы делали для WhatsApp. Хочешь — сделаем!
